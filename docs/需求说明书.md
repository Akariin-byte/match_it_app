## 1. 项目概述

### 1.1 背景与目标

MATCHit 是一款**兴趣搭子匹配**应用，帮助用户按兴趣领域（桌游、美食、运动）和活动强度档位，发现合适的组队活动。核心差异化能力是基于**捏脸特征标签**的用户—发帖人相似度匹配，并结合互动量、活跃度做推荐排序。

产品采用**「先体验、后绑定」**注册策略（参考小红书）：用户可先以游客身份浏览，再绑定手机号升级为正式账号，降低首次使用门槛。

### 1.2 产品范围（当前版本）

| 范围 | 说明 |
|------|------|
| 客户端 | Flutter 跨平台应用（Web / Android / iOS / Desktop），主入口 `lib/main.dart` |
| 服务端 | Go + Gin REST API，PostgreSQL 16 + pgvector |
| 基础设施 | Docker Compose 本地 Postgres；PowerShell 一键启动脚本 |

### 1.3 不在当前版本范围

- Apple / Google 第三方登录（UI 占位，未实现）
- 真实 IM 私聊、申请加入的后端持久化
- Redis 生产部署（客户端仅有内存缓存模拟）
- 向量相似检索 HTTP API（SQL 已设计，Handler 未接入）
- 支付、信用分变更、内容审核等运营能力

## 2. 用户角色

| 角色 | 描述 | 识别方式 |
|------|------|----------|
| **游客** | 未绑定手机号的临时用户，可浏览 Feed | JWT 中 `isGuest=true` |
| **正式用户** | 已绑定大陆手机号的注册用户 | JWT 中 `isGuest=false`，`phone` 有值 |
| **活动主理人** | 发布组队活动的用户（概念角色，发帖 API 已具备） | 帖子 `host_*` 字段 |
| **平台运营** | 置顶官方活动 | 帖子 `isPinned` / `pinPriority` |

## 3. 术语定义

| 术语 | 定义 |
|------|------|
| 领域（Area） | 兴趣分类：`BoardGames`（桌游）、`Food`（美食）、`Sport`（运动） |
| 强度档位 | 用户/活动硬核程度，取值 0 / 25 / 50 / 75 / 100，对应新手→大神 |
| 捏脸特征 | 描述用户性格/偏好的标签集合，如「策略思维」「社交型」 |
| 特征向量 | 将捏脸标签编码为 18 维 L2 归一化向量，用于相似度计算 |
| 匹配分 | 用户向量与发帖人向量的余弦相似度，映射为 0–100 分 |
| Tab | Feed 频道：推荐、匹配、附近、桌游 |

## 4. 功能需求

### 4.1 用户认证模块（AUTH）

#### AUTH-01 游客登录

| 项 | 说明 |
|----|------|
| 优先级 | P0 |
| 实现状态 | ✅ 已实现（后端 + 客户端） |
| 描述 | 用户无需密码，一键创建临时身份并获取 JWT |
| 触发 | 登录页点击「先逛逛，暂不登录」 |
| 后端 | `POST /api/v1/auth/guest` |
| 业务规则 | 自动生成 `openid=guest_{uuid}`；`is_guest=true`；游客 Token 有效期默认 720 小时 |

#### AUTH-02 手机号绑定

| 项 | 说明 |
|----|------|
| 优先级 | P0 |
| 实现状态 | ✅ 已实现（后端 + 客户端） |
| 描述 | 游客绑定手机号后升级为正式用户，身份 ID 不变 |
| 触发 | 登录页「Continue with Phone」；或 Feed 页游客横幅「绑定手机」 |
| 后端 | `POST /api/v1/auth/bind-phone`（需 Bearer 游客 Token） |
| 业务规则 | 手机号须为大陆 11 位（`^1[3-9]\d{9}$`）；全局唯一；仅 `is_guest=true` 可绑定；绑定后重发 JWT（`isGuest=false`） |
| 错误码 | 400 无效手机号；403 非游客；409 已注册或手机号占用 |

#### AUTH-03 身份校验中间件

| 项 | 说明 |
|----|------|
| 优先级 | P0 |
| 实现状态 | ✅ 后端已实现 |
| 描述 | Gin 中间件解析 JWT，向上下文注入 `user_id`、`openid`、`is_guest` |
| 用途 | 保护 `GET /me`、`POST /posts`、`POST /bind-phone` 等接口 |

#### AUTH-04 第三方登录

| 项 | 说明 |
|----|------|
| 优先级 | P2 |
| 实现状态 | ❌ 未实现（UI 占位） |
| 描述 | Apple / Google OAuth 登录 |

#### AUTH-05 传统注册（Continue 按钮）

| 项 | 说明 |
|----|------|
| 优先级 | P2 |
| 实现状态 | ⚠️ 部分实现 |
| 描述 | 填写姓名 + 账号后直接进区域选择，**不调用后端**，无 JWT |

|----|------|

## 5. 业务流程

### 5.1 先体验后绑定（主流程）

```
打开 App
  → 点击「先逛逛，暂不登录」
  → POST /auth/guest（获取 Token + userId）
  → 区域选择（领域 + 强度）
  → 首页 Feed（橙色游客横幅）
  → [可选] 点击「绑定手机」→ POST /auth/bind-phone
  → 横幅变绿，Token 更新为正式用户
```

### 5.2 手机号直达正式用户

```
打开 App
  → 输入手机号
  → 点击「Continue with Phone」
  → POST /auth/guest → POST /auth/bind-phone
  → 区域选择 → 首页（绿色已绑定横幅）
```

### 5.3 浏览帖子

```
首页 Feed
  → 切换 Tab / 搜索
  → 点击卡片
  → 帖子详情
  → [可选] 申请加入 / 私聊（当前为 Mock）
```

## 7. 接口规格摘要

**Base URL**：`http://localhost:8080`（Flutter 可通过 `--dart-define=API_BASE_URL=` 配置）

| 方法 | 路径 | 认证 | 说明 |
|------|------|------|------|
| GET | `/health` | 无 | 健康检查 + 帖子数 |
| POST | `/api/v1/auth/guest` | 无 | 游客登录 |
| POST | `/api/v1/auth/bind-phone` | Bearer（游客） | 绑定手机号 |
| GET | `/api/v1/me` | Bearer | 当前用户身份 |
| GET | `/api/v1/posts` | 无 | 帖子列表 `?area=&tab=` |
| GET | `/api/v1/posts/:id` | 无 | 帖子详情 |
| POST | `/api/v1/posts` | Bearer | 创建/更新帖子 |
| POST | `/api/v1/seed` | 无 | 导入种子数据 |

## 12. 强度档位过滤规则

用户强度分值与可见帖子 `hardcoreScore` 的对应关系：

| 用户 intensityScore | 可见帖子 hardcoreScore |
|---------------------|------------------------|
| 0（新手） | < 25 |
| 25（普通） | [25, 50) |
| 50（进阶） | [50, 75) |
| 75（硬核） | [75, 90) |
| 100（大神） | ≥ 90 |

> 「匹配」Tab **不应用**上述强度过滤。

|------|------|
| DATABASE_URL | Postgres 连接串 |
| JWT_SECRET | JWT 签名密钥（必填） |
| PORT | 默认 8080 |
| JWT_GUEST_TTL_HOURS | 游客 Token 有效期，默认 720 |
| JWT_MEMBER_TTL_HOURS | 正式用户 Token 有效期，默认 168 |

### 14.2 客户端启动

```bash
flutter pub get
flutter run -d chrome --dart-define=API_BASE_URL=http://localhost:8080
```
